Settings - Profile

Profiles are setup here. The last selected profile is automatically loaded on startup of Axis3.

There are 2 types of profile, local and remote.

 

Selecting Already Created Profile

To select an already created profile, pick the desired profile from the Name dropdown box.

•Select - Click here to load the selected profile.

•Delete - Click here to delete the selected profile.

*Note* Default profile can’t be deleted!

•Re-Load - Click here to re-build the database for the selected profile.

 

Profile Creation

To create a profile, simply place the cursor in the Name field and type in a new name.

Next step is to select a local profile or remote.

 

Local Profile Setup

To make it a local profile, select the check box Local Profile. This will enable the Browse button and disable remote profile options.

Click Browse button to select the spheretables.scp to b loaded.

*Note* Axis3 will read spheretable.scp and load all files from the [RECOURSE] section to load all files the same as sphereserver would.

Click Load button to save your settings and load this profile.

 

Remote Profile Setup
To make it a remote profile, Un select the check box Local Profile. This will enable the all remote profile options, and disable the Browse button. *This is the default option.

Remote profiles require to connect to a running sphereserver with staff account (PLEVEL 2+).

In the IP Address field, enter the IP address of the remote server.

Enter the port in the Port field.

Enter your staff account name and password in the Account and Password fields.

Click Load button to save your settings and load this profile.

 

Remote Profile Server Side Setup
The server side setup is simple. Place the provided command line program ‘AxisDBCreator.exe’ in the same folder as ‘sphere.exe’.

Place the following script in your script folder.

*Note* You might need to edit the spheretables.scp path.

 

[FUNCTION f_axisdb]

local.path = scripts\spheretables.scp

local.return = <sysspawn AxisDBCreator.exe <local.path>>

SERV.LOG AxisDB: <local.return>

 

Now you can run this function manually by typing in the console f_axisdb and this will create the database for users to download.

You can also place this function inside another timed function or server trigger to run it periodically.

*Note* If you updated a single script, you can run the same function but with a path to this single script. This will update the database instead of rebuilding it.
